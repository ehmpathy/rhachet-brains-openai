# blueprint: product

## summary

implement tool use support for brain atoms via `plugs.tools`.

when tools are plugged:
1. convert tool definitions to openai function format
2. pass to responses api via `tools` parameter
3. parse `function_call` items from response output
4. extract output and calls independently (both can be present)
5. accept `BrainPlugToolExecution[]` as prompt for continuation
6. skip `response_format` (mutually exclusive with tools)

## key implementation details

### 1. cast operations at openai boundary

```ts
// out: convert tool definitions
const openaiFunctionTools = askInput.plugs.tools.map((definition) =>
  castIntoOpenaiFunctionTool({ definition }),
);

// out: convert tool results for continuation
const currentInput = isToolExecutionArray
  ? askInput.prompt.map((execution) =>
      castIntoOpenaiFunctionCallOutput({ execution }),
    )
  : [{ role: 'user', content: fullPrompt }];

// in: extract invocations from response
const calls = functionCallItems.length > 0
  ? { tools: functionCallItems.map((item) => castFromOpenaiFunctionCall({ item })) }
  : null;
```

### 2. response_format and tools are mutually exclusive

```ts
// when tools present: use tools, skip response_format
// when no tools: use response_format for structured output
const hasTools = askInput.plugs?.tools && askInput.plugs.tools.length > 0;

await openai.responses.create({
  model: config.model,
  input: inputMessages,
  ...(hasTools && {
    tools: openaiFunctionTools,
  }),
  ...(!hasTools && isObjectSchema && {
    text: {
      format: { type: 'json_schema', ... },
    },
  }),
});
```

### 3. independent extraction pattern

```ts
// extract tool calls (always, regardless of text presence)
const functionCallItems = response.output.filter(
  (item): item is OpenAI.Responses.ResponseFunctionToolCall =>
    item.type === 'function_call',
);
const calls = functionCallItems.length > 0
  ? { tools: functionCallItems.map((item) => castFromOpenaiFunctionCall({ item })) }
  : null;

// extract message output (always, regardless of tool_use presence)
const messageItem = response.output.find((item) => item.type === 'message');

// try to parse output from text, with graceful fallback
const output = (() => {
  if (!messageItem) return null;
  const textContent = messageItem.content.find((c) => c.type === 'output_text');
  if (!textContent) return null;
  try {
    return askInput.schema.output.parse(JSON.parse(textContent.text));
  } catch {
    // parse failed — if tool calls present, this is expected (partial response)
    if (calls) return null;
    throw new Error('structured output parse failed with no tool calls');
  }
})();

// both output AND calls can be present simultaneously
// e.g., "Let me look that up for you" + function_call
return new BrainOutput({ output, calls, ... });
```

### 4. tool exchanges use standard exchange pattern

```ts
// tool exchanges serialize the same as message exchanges
// no special-case code needed

const exchangeInput = isToolResultContinuation
  ? JSON.stringify(askInput.prompt)  // tool_result blocks as JSON
  : (askInput.prompt as string);     // plain string

const exchangeOutput = functionCallItems.length > 0
  ? JSON.stringify(response.output)  // full content with function_call items
  : outputText;                      // plain structured output
```
