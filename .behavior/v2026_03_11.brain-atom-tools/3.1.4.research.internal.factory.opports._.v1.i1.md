# research: internal factory opportunities

## the constraint

**bottleneck**: api call latency (~2-5s per call)

this is inherent to integration tests against external apis. unavoidable for real verification.

**mitigation**: use gpt-4o-mini (fast model) for tests — already in place.

| queue | wait time | controllable? |
|-------|-----------|---------------|

| test execution | sequential | yes (parallelizable) |

**bottleneck is external** — api latency dominates, internal queues are minimal.

## small batches

**current capability**:
- can run individual test files: `npm run test:integration -- genBrainAtom.integration.test.ts`
- can run specific test blocks via jest's `-t` flag
- feedback available per test (~30s)

**smallest verifiable unit**: single integration test case.

**opportunity**: structure tests as independent usecases so each can pass/fail independently. already in place via given/when/then pattern.
