## issue found: inconsistent z.toJSONSchema usage

### what i found

the codebase has an extant `asJsonSchema` utility:

```ts
// src/infra/schema/asJsonSchema.ts
export const asJsonSchema = (input: { schema: z.ZodSchema }): object => {
  return z.toJSONSchema(input.schema, { target: 'openAi' });
};
```

`genBrainRepl.ts` correctly uses this utility:

```ts
// src/domain.operations/repls/genBrainRepl.ts:130
const outputSchema = asJsonSchema({
  schema: input.schema.output,
});
```

but in the new code, `z.toJSONSchema` is called directly without the `{ target: 'openAi' }` option:

```ts
// src/domain.operations/atoms/genBrainAtom.ts:107
const jsonSchema = z.toJSONSchema(askInput.schema.output);

// src/infra/cast/castIntoOpenaiFunctionTool.ts:22
parameters: z.toJSONSchema(input.definition.schema.input) as {...}
```

### why this matters

the `{ target: 'openAi' }` option affects JSON schema generation to match OpenAI's expected format. the inconsistency could cause subtle compatibility issues.

### fix applied

updated both files to use `asJsonSchema`:

**genBrainAtom.ts:**
```ts
import { asJsonSchema } from '../../infra/schema/asJsonSchema';
// ...
const jsonSchema = asJsonSchema({ schema: askInput.schema.output });
```

**castIntoOpenaiFunctionTool.ts:**
```ts
import { asJsonSchema } from '../schema/asJsonSchema';
// ...
parameters: asJsonSchema({ schema: input.definition.schema.input }) as {...}
```
