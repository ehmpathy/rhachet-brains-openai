#### boundary.1 gap analysis

**criteria**: "when invocation.slug does not match any plugged tool, error is surfaced"

**blueprint status**: not explicitly covered in test cases or codepath

**analysis**:
- this is a caller-side concern, not atom-side
- brain returns whatever openai returns as function_call.name
- if brain hallucinates a non-existent tool slug, caller handles via toolDict lookup
- atom's job is to pass through invocations, not validate them

**resolution options**:
1. add validation in atom (atom responsibility)
2. defer to caller (caller responsibility)
3. flag as out of scope (not atom's concern)

**recommendation**: this boundary is caller-side. the atom faithfully returns what openai provides. callers use `asBrainPlugToolDict` to handle unknown slugs. no blueprint change needed.
