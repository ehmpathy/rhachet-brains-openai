### Q1: how exactly does openai format tool definitions?

**can research?** yes

**answer**:

openai tools are defined with:
```json
{
  "type": "function",
  "name": "get_weather",
  "description": "get current temperature for a location",
  "parameters": {
    "type": "object",
    "properties": {
      "location": { "type": "string", "description": "city and country" }
    },
    "required": ["location"],
    "additionalProperties": false
  },
  "strict": true
}
```

key points:
- `type: "function"` at top level
- `strict: true` ensures schema adherence
- `additionalProperties: false` required for strict mode
- parameters use JSON Schema format

**source**: [OpenAI Function Call Guide](https://platform.openai.com/docs/guides/function-calling)

**verdict**: ✅ answered. our zod → JSON Schema approach will work.

### Q2: how are tool calls returned in response?

**can research?** yes

**answer**:

responses api returns `output` array with items. tool calls appear as items with:
- `type: "function_call"`
- `id`: correlation id for result submission
- `name`: function name
- `arguments`: JSON string of parsed args

**source**: [OpenAI Responses API Migration](https://platform.openai.com/docs/guides/migrate-to-responses)

**verdict**: ✅ answered. maps to `BrainPlugToolInvocation { exid, slug, input }`.

### Q3: how are tool results submitted?

**can research?** yes

**answer**:

for responses api, tool results are submitted as input items with:
- `type: "function_call_output"`
- `call_id`: matched to the original function call id
- `output`: string result

this is different from chat completions which uses role: "tool" messages.

**source**: [OpenAI Responses API](https://platform.openai.com/docs/api-reference/responses)

**verdict**: ✅ answered. prompt injection won't work correctly. need explicit tool result input.

**action required**: update vision to note that tool result continuation needs explicit support, not prompt injection.

### Q4: does output schema work alongside tools?

**can research?** yes

**answer**:

**NO** — they serve different purposes and should not be combined:
- `response_format` = for structured final output to user
- `tools` = for function call to external systems

openai docs recommend one or the other based on use case.

**source**: [OpenAI Structured Outputs](https://developers.openai.com/api/docs/guides/structured-outputs/)

**verdict**: ✅ answered.

**implication**: when tools are plugged, we should NOT use response_format. the wisher's "w/ and w/o output schema" likely means:
- test tool use WITHOUT output schema (tools mode)
- test inference WITH output schema (current behavior, no tools)

NOT "test tools with and without output schema simultaneously".

**action required**: update vision to clarify this mutual exclusivity.
