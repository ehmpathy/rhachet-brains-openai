| rule.forbid.positional-args | all inputs are named objects | ✓ adheres |

**why it holds**:

1. **(input, context) pattern**: the ask() method signature shows `(askInput, context)` where askInput is a typed object with named properties. the extant pattern is preserved.

| standard | blueprint adherance | verdict |
|----------|---------------------|---------|

| rule.require.narrative-flow | codepath tree shows linear flow | ✓ adheres |
| rule.prefer.early-returns | function_call check returns early | ✓ adheres |

**why it holds**:

1. **no else branches**: the blueprint shows conditional spread syntax:
   ```ts

   ...(!hasTools && isObjectSchema && { text: { format: {...} } }),
   ```

2. **narrative flow**: the codepath tree shows linear progression:
   - tool definition conversion
   - prompt type branch
   - api call

   each step follows the next without nested conditionals.

3. **early returns**: the blueprint shows that when function_call items are present, the code returns early with `output: null, calls: {...}`. the message extraction path only runs when no function_calls exist.
