## implementation notes

### key discoveries

1. **function_call_output.id requires fc_ prefix**: openai requires the `id` field to start with `fc_` prefix, separate from `call_id` which references the original function_call
2. **episode reconstruction for function_calls**: to continue with tool results, prior exchanges with function_calls must be reconstructed as response items (not plain text)
3. **system instruction for JSON output**: when tools are plugged, `response_format` is not available, so we add system instruction that requests JSON output to match the schema
4. **model may use tools when not needed**: gpt-4o-mini is eager to use tools (e.g., calculator for 2+2), which is valid behavior - tests accept either direct answer or tool use
5. **plain text response on tool errors**: model may respond with plain text after it receives a tool error, which does not parse as structured output - this is valid behavior
